## üìã Submission Process

### Step 1: Public Beta Test (REQUIRED)

Before submitting to Hosted Games, you **must** run a public beta on the Choice of Games forum.

1. **Create Forum Account**: https://forum.choiceofgames.com/
2. **Post in Beta Testing Section**: https://forum.choiceofgames.com/c/beta-testing/5
3. **Include in Your Post**:
   - Game title: "Polly's Wingscroll: The First Thread"
   - Brief description (2-3 paragraphs)
   - Link to playable beta
   - Request for feedback on:
     - Story flow and pacing
     - Character development
     - Choice meaningfulness
     - Technical issues/bugs
     - Typos and grammar

4. **Beta Test Duration**: Minimum 2-4 weeks
5. **Gather Feedback**: Respond to comments, fix bugs, improve based on feedback

### Step 2: Prepare Submission Files

**What to Send:**
- All `.txt` files from `choicescript_game/scenes/` folder
- `startup.txt` file
- `choicescript_stats.txt` file (if customized)

**How to Package:**
1. Create a ZIP file containing all scene files
2. Name it: `pollys-wingscroll-scenes.zip`

### Step 3: Submit to Hosted Games

**Email to**: hg-submissions@choiceofgames.com

**Subject Line**: "Hosted Games Submission - Polly's Wingscroll"

**Email Body Template**:

```
Dear Hosted Games Team,

I am submitting "Polly's Wingscroll: The First Thread" for consideration as a Hosted Game.

Game Details:
- Title: Polly's Wingscroll: The First Thread
- Author: [Your Name]
- Word Count: ~40,000 words
- Estimated Playtime: 45-90 minutes
- Number of Endings: 14
- Forum Beta Thread: [LINK TO YOUR BETA THREAD]

Game Description:
Polly's Wingscroll is a choice-based narrative set in Avalon Academy, a living pocket dimension where students learn collaborative dimensional magic. Players navigate relationships with legendary mages, choose between three expedition paths (Singing Dunes, Verdant Tithe, or Rune Glacier), and shape their destiny through meaningful choices. The game emphasizes collaboration over competition, with 14 unique endings ranging from legendary mastery to humble determination.

The game has completed a [X-week] public beta test with [Y] participants providing feedback, which has been incorporated into the final version.

Attached:
- pollys-wingscroll-scenes.zip (all game files)

Thank you for your consideration.

[Your Name]
[Your Contact Information]
```

## üìä Expected Timeline

### After Submission:

1. **Initial Response**: 1-4 weeks
2. **Content Review**: 2-6 weeks
   - They'll check for content guidelines compliance
   - May request changes or clarifications
3. **Contract Offer**: If approved
4. **Publication Preparation**: 2-8 weeks
   - Cover art commission
   - App store setup
   - Final testing
5. **Release**: Usually 4-6 months from submission to publication

### Potential Issues to Watch:

- **Content Guidelines**: Make sure your game meets their content policy
- **Technical Requirements**: All scenes must be properly formatted
- **Completion**: Game must be 100% complete (no "to be continued")
- **Length**: 30,000+ words minimum (we meet this)

## üí∞ Financial Information

### Revenue Share:
- **Standard Split**: 25% royalties to author, 75% to Choice of Games LLC
- **Payment**: Quarterly payments when earnings exceed $20 threshold
- **Platforms**: iOS, Android, Steam (if 150,000+ words), Web

### Pricing:
- Hosted Games typically price at $2.99-$4.99
- Choice of Games sets final price
- Free demo (first few chapters) available to players

### Estimated Earnings:
- Varies widely based on quality, marketing, reviews
- Successful Hosted Games can earn $500-$5,000+ over their lifetime
- Top performers can earn significantly more
- Our game (~40k words) would likely fall in mid-range category

## üéØ Improving Your Chances

### Before Submission:

1. **Extensive Beta Testing**
   - Get 20+ beta testers if possible
   - Address all major bugs and issues
   - Polish based on feedback

2. **Professional Presentation**
   - Fix all typos and grammar errors
   - Ensure consistent tone and voice
   - Check all achievement triggers work

3. **Meaningful Choices**
   - Make sure choices actually matter
   - Avoid false choices when possible
   - Ensure stats affect outcomes

4. **Replay Value**
   - Multiple paths should feel different
   - Endings should reflect choices meaningfully
   - Players should want to replay

### Marketing (After Publication):

- Engage with the CoG community on forums
- Share on social media
- Request honest reviews
- Respond to player feedback
